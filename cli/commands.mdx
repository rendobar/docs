Every `rb` command, every flag, every exit code. Look things up here. Use the [overview](/cli/overview) or the [how-tos](/cli/ci-cd) if you want narrative.

## Global flags

These are accepted by `rb ffmpeg`. Other commands ignore them.

| Flag | Type | Default | Effect |
|---|---|---|---|
| `--json` | bool | `false` | Print the full job result as JSON to stdout. Suppresses progress display |
| `--url-only` | bool | `false` | Print only the result URL to stdout. Suppresses progress display |
| `--quiet` | bool | `false` | Print nothing. Use the exit code |
| `--no-wait` | bool | `false` | Submit the job and exit. Prints the job ID to stdout |
| `--timeout N` | int | `120` | Max execution time in seconds. Range 1–900 |

`--json` and `--url-only` are mutually exclusive in practice — the first one parsed wins. `--quiet` overrides both.

## Exit codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | Job ran but failed, or a generic runtime error |
| `2` | User error: not authenticated, invalid arguments, port in use, insufficient credits |
| `130` | Cancelled via `Ctrl+C` (SIGINT). The CLI attempts to cancel the in-flight job before exiting |

Use `--quiet` plus the exit code in shell scripts that only care about success/failure.

## `rb doctor`

Run diagnostic checks against the install, the API, and the local auth state.

```
rb doctor [--json] [--fix]
```

Eight checks run in order: CLI version vs latest release, install method, OS/arch, update cache, `api.rendobar.com` reachability, auth state, macOS quarantine bit, GitHub API rate limit.

| Flag | Effect |
|---|---|
| `--json` | Print results as JSON instead of the human-readable table |
| `--fix` | Attempt to fix what can be fixed. Today: removes the macOS quarantine bit from the `rb` binary |

Exits `0` when all checks pass, `1` if any failed (look at the `fix:` lines). Warnings do not change the exit code.

```bash
rb doctor
```

```
rb 1.0.0 doctor

  [ok  ] version: 1.0.0 (latest)
  [ok  ] install method: standalone binary (linux-x64)
  [ok  ] OS/arch: linux x64
  [ok  ] update cache: cached latest=1.0.0 (3h old)
  [ok  ] api.rendobar.com: HTTP 200
  [ok  ] auth: oauth token present (not verified)
  [ok  ] macOS quarantine: N/A
  [ok  ] GitHub rate limit: 58/60 remaining

All checks passed.
```

## `rb ffmpeg`

Run FFmpeg in the cloud. Pass real FFmpeg arguments after the subcommand name.

```
rb ffmpeg [--json | --url-only | --quiet] [--no-wait] [--timeout N] <ffmpeg args>
```

Global flags (`--json`, `--url-only`, `--quiet`, `--no-wait`, `--timeout N`) are stripped by name before the rest is passed to the cloud FFmpeg invocation.

Local input files (`-i ./clip.mp4`) are uploaded automatically before submission and rewritten to short-lived URLs. Remote inputs (`-i https://...`) are passed through as-is.

The output filename in your command (the trailing positional argument) is the local path the result is downloaded to. With `--no-wait` no download happens — only the job ID is printed.

`--timeout N` sets the server-side execution timeout in seconds (default `120`, range 1–900). The other global flags above all apply.

```bash
# Remote input → local output (default)
rb ffmpeg -i https://cdn.example.com/clip.mp4 -vf scale=1280:720 -c:v libx264 -crf 23 out.mp4

# Local input (uploaded automatically)
rb ffmpeg -i ./local.mp4 -c:v libx265 -preset medium -crf 28 out.mp4

# Submit only, get the job ID, exit
rb ffmpeg --no-wait -i ./local.mp4 -c:v libx264 out.mp4

# CI assertion: render must succeed, no output otherwise
rb ffmpeg --quiet -i in.mp4 -c:v libx264 out.mp4 || exit 1

# Pipe the result URL to wget
URL=$(rb ffmpeg --url-only -i in.mp4 -c:v libx264 out.mp4)
wget "$URL" -O out.mp4
```

Job-failure exits with `1` and writes the API error message to stderr. Insufficient credits exits with `2` and prints a top-up link.

## `rb login`

Authenticate the CLI. Default flow opens a browser; `--key` is the non-interactive fallback.

```
rb login [--key <api-key>]
```

`--key <api-key>` skips the browser flow and saves the given key after verifying it against the API. Without `--key`, the browser flow listens on `http://127.0.0.1:14832/callback` and times out after 120 seconds.

See [Authentication](/cli/authentication) for credential storage paths and the env var override.

```bash
# Browser flow
rb login

# CI / SSH
rb login --key rb_live_xxx
```

## `rb logout`

Sign out. Revokes the OAuth refresh token (best-effort) and deletes the credentials file.

```
rb logout
```

No flags. No-op when not signed in.

## `rb update`

Check for and install the latest release.

```
rb update
```

No flags. Behaviour:

1. Fetches the latest non-prerelease tag from `github.com/rendobar/cli/releases`.
2. If you're already on the latest, exits with `Already on the latest version (X.Y.Z).`
3. Downloads the release archive for your platform, verifies the SHA-256 against `checksums.txt`.
4. Atomically replaces the running binary; rolls back if the new binary fails to start.
5. Writes a log entry to `~/.rendobar/update.log`.

In dev mode (running from source via `bun run`), `rb update` prints the install command instead of self-replacing.

Exits `1` on any failure (network, checksum, write permission). The old binary is preserved on rollback.

## `rb whoami`

Print the current authenticated identity.

```
rb whoami
```

No flags. Hits `/orgs/current` with the saved credential and prints org, plan, and balance:

```
  Org       Acme Corp
  Plan      Pro
  Balance   $42.85
```

Exits `2` if not authenticated, `1` if the API call fails.
