Point any MCP-compatible agent at `https://api.rendobar.com/mcp` and it can submit jobs, poll status, upload files, and check the balance through five typed tools — without writing HTTP code.

## Connect

<CodeGroup>

```json Claude Desktop / Code
{
  "mcpServers": {
    "rendobar": {
      "url": "https://api.rendobar.com/mcp",
      "headers": {
        "Authorization": "Bearer rb_live_YOUR_KEY"
      }
    }
  }
}
```

```text Custom agent
Transport: Streamable HTTP
URL:       https://api.rendobar.com/mcp
Header:    Authorization: Bearer rb_live_YOUR_KEY
```

</CodeGroup>

## Tools

| Tool | What it does |
|---|---|
| `submit_job` | Submit a job. Description dynamically lists available job types from the registry |
| `get_job` | Status, progress, result, output URL |
| `list_jobs` | Recent jobs with optional status / type filter |
| `get_account` | Balance, plan, active job count, rate-limit usage |
| `upload_media` | Get a temporary upload endpoint for a local file |

Detailed parameters: [Tools reference](/mcp/tools-reference).

<Tip>
The `submit_job` description enumerates available job types at runtime, so agents discover capabilities without a separate discovery tool.
</Tip>

## Authentication

API key in the `Authorization` header — same key as REST. MCP clients with native OAuth 2.1 can use Rendobar's authorization server instead:

1. **Register** — client dynamically registers with the authorization server
2. **Authorize** — user grants consent in their browser (one-time)
3. **Token** — client receives an access token scoped to the user's organization
4. **Operate** — subsequent tool calls use the token automatically

API keys are simpler and recommended for non-interactive integrations.

## Example: agent submits a render

```
1. get_account()
   → { balance: "$10.50", plan: "pro", limits: { activeJobs: "0/25" } }

2. submit_job({
     type: "raw.ffmpeg",
     inputs: { "video.mp4": "https://example.com/video.mp4" },
     params: { command: "ffmpeg -i video.mp4 -vf scale=1280:720 output.mp4" }
   })
   → { jobId: "job_x1y2z3", status: "dispatched", estimatedCost: "$0.05" }

3. get_job({ jobId: "job_x1y2z3" })
   → { status: "complete", cost: "$0.05", outputUrl: "https://..." }
```

## Example: agent uploads a local file

```
1. upload_media({ filename: "video.mp4" })
   → { uploadEndpoint: "https://api.rendobar.com/uploads?filename=video.mp4",
       method: "POST",
       instructions: "..." }

2. (Agent or user POSTs the file to uploadEndpoint)

3. submit_job({
     type: "raw.ffmpeg",
     inputs: { "video.mp4": "https://api.rendobar.com/uploads/dl/..." },
     params: { command: "ffmpeg -i video.mp4 -vf scale=640:480 output.mp4" }
   })
```

## Rate limits + billing

MCP tool calls share the same rate limits and credit model as REST. Each `submit_job` deducts on completion. Limits are per-organization.
