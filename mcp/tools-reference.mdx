Rendobar exposes 5 tools through its MCP server at `api.rendobar.com/mcp`. All tools require authentication via API key or OAuth token.

## Job submission

<ParamField body="submit_job" type="tool">
  Submit any media processing job. The tool description dynamically lists all available job types from the registry.

  **Parameters:**
  - `type` (string, required) -- Job type from the registry (e.g., `raw.ffmpeg`) or use the dynamic list in the tool description
  - `inputs` (Record\<string, string\>, required) -- Map of input name to URL. For FFmpeg jobs, keys should match filenames referenced in your command
  - `params` (Record\<string, unknown\>, optional) -- Type-specific parameters. For `raw.ffmpeg`: `{ command: "..." }`
  - `idempotencyKey` (string, optional) -- Prevents duplicate jobs on retry

  **Response:** `{ jobId, status, estimatedCost, timeoutSeconds }`
</ParamField>

## Job monitoring

<ParamField body="get_job" type="tool">
  Get job status, progress, results, and output URL by job ID.

  **Parameters:**
  - `jobId` (string, required) -- Job ID returned by `submit_job`

  **Response includes:**
  - `status` -- waiting, dispatched, running, complete, failed, cancelled
  - `progress` -- fraction complete (e.g., "3/7 steps") when running
  - `cost` -- formatted cost (e.g., "$0.05") when complete
  - `outputUrl` -- download URL when complete
  - `output` -- metadata (format, resolution, duration, file size) when complete
  - `error` -- error details when failed
</ParamField>

<ParamField body="list_jobs" type="tool">
  List recent jobs with optional filters.

  **Parameters:**
  - `status` (enum, optional) -- Filter: waiting, dispatched, running, complete, failed, cancelled
  - `type` (string, optional) -- Filter by job type
  - `limit` (number, 1-50, default 10) -- Number of jobs to return

  **Response:** `{ jobs: [{ id, type, status, createdAt, cost }], total }`
</ParamField>

## Account

<ParamField body="get_account" type="tool">
  Check credit balance, plan limits, active jobs, and rate limit usage. Call before submitting expensive jobs to verify affordability.

  **Parameters:** None

  **Response:**
  - `balance` -- formatted balance (e.g., "$10.50")
  - `balanceNanodollars` -- raw balance in nanodollars
  - `plan` -- current plan name (free or pro)
  - `limits` -- active jobs count vs max, max file size, job timeout
  - `costGuidance` -- brief pricing guide for common job types
</ParamField>

## File upload

<ParamField body="upload_media" type="tool">
  Get upload instructions for a local file. Returns the upload endpoint URL and a curl command. After uploading, the response contains a `downloadUrl` to use as input in `submit_job`.

  **Parameters:**
  - `filename` (string, required) -- Filename with extension (e.g., `video.mp4`)
  - `contentType` (string, optional) -- MIME type. Auto-detected from extension if omitted.

  **Response:**
  - `uploadEndpoint` -- URL to POST the file to (with same API key auth)
  - `method` -- POST
  - `contentType` -- detected MIME type
  - `maxFileSize` -- plan-based size limit
  - `expiresIn` -- 24 hours
  - `instructions` -- step-by-step upload guide

  Files are stored in ephemeral R2 storage and auto-deleted after 24 hours.
</ParamField>
