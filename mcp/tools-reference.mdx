Reference for the five tools exposed by the Rendobar MCP server at `https://api.rendobar.com/mcp`. All tools require an API key or OAuth token. For setup, see [MCP overview](/mcp/overview).

## `submit_job`

Submit any job type. The tool description dynamically lists available types from the registry.

<ParamField body="type" type="string" required>
  Job type from the registry (e.g. `raw.ffmpeg`).
</ParamField>

<ParamField body="inputs" type="Record<string, string>" required>
  Map of input name to URL. For FFmpeg jobs, keys should match filenames referenced in your command.
</ParamField>

<ParamField body="params" type="Record<string, unknown>">
  Type-specific parameters. For `raw.ffmpeg`: `{ command: "ffmpeg -i ..." }`.
</ParamField>

<ParamField body="idempotencyKey" type="string">
  Replay-safe key. Resubmitting the same key returns the original job.
</ParamField>

**Response:** `{ jobId, status, estimatedCost, timeoutSeconds }`

## `get_job`

Status, progress, result, output URL by job ID.

<ParamField body="jobId" type="string" required>
  Job ID returned by `submit_job`.
</ParamField>

**Response includes:**

- `status` — `waiting | dispatched | running | complete | failed | cancelled`
- `progress` — fraction complete while `running` (e.g. `"3/7 steps"`)
- `cost` — formatted cost (e.g. `"$0.05"`) when `complete`
- `outputUrl` — signed download URL when `complete`
- `output` — metadata (format, resolution, duration, file size) when `complete`
- `error` — `{ code, message }` when `failed`

## `list_jobs`

Recent jobs with optional filters.

<ParamField body="status" type="enum">
  Filter by status: `waiting | dispatched | running | complete | failed | cancelled`.
</ParamField>

<ParamField body="type" type="string">
  Filter by job type.
</ParamField>

<ParamField body="limit" type="integer" default="10">
  Number of jobs to return. Range 1–50.
</ParamField>

**Response:** `{ jobs: [{ id, type, status, createdAt, cost }], total }`

## `get_account`

Balance, plan, active job count, rate-limit usage. Call before expensive jobs.

**Parameters:** none.

**Response:**

- `balance` — formatted (e.g. `"$10.50"`)
- `balanceNanodollars` — raw integer
- `plan` — `free | pro`
- `limits` — active jobs vs max, max file size, job timeout
- `costGuidance` — short pricing guide for common job types

## `upload_media`

Get a temporary upload endpoint for a local file. After upload, the response gives a `downloadUrl` to use as input.

<ParamField body="filename" type="string" required>
  Filename with extension (e.g. `video.mp4`).
</ParamField>

<ParamField body="contentType" type="string">
  MIME type. Auto-detected from the extension if omitted.
</ParamField>

**Response:**

- `uploadEndpoint` — URL to `POST` the file to (auth with the same API key)
- `method` — `POST`
- `contentType` — detected MIME type
- `maxFileSize` — plan-based size cap
- `expiresIn` — 24 hours
- `instructions` — step-by-step upload guide

Files live in ephemeral R2 storage and auto-delete after 24 hours.
