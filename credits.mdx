Rendobar uses a credit-based billing system. Every job costs credits based on either a flat rate or the duration of the input/output media. Your balance is denominated in dollars.

## How credits work

### Credit flow

<Steps>

<Step title="Pre-check">
  Before a job is dispatched, Rendobar checks that your organization can afford the estimated cost. This does not deduct credits.
</Step>

<Step title="Execution">
  The job runs on the execution provider.
</Step>

<Step title="Debit">
  On completion, the actual cost is calculated and credits are atomically deducted from your balance.
</Step>

<Step title="Refund">
  If a job fails or is cancelled, no credits are deducted. Failed jobs never cost anything.
</Step>

</Steps>

## Credit packs

Purchase additional credits at any time: $10 / $25 / $50 / $100 or custom (minimum $10).

Pro subscribers receive a **20% bonus** on every purchase. Buy $25, get $30 in credits. Buy $100, get $120.

Credits never expire.

## Cost estimates

Before submitting a job, Rendobar checks whether your balance covers the estimated cost:

| Job Type | Cost Model | Typical Cost |
|----------|-----------|-------------|
| raw.ffmpeg | Per compute time | ~$0.05/min |
| extract.metadata | Flat rate | $0.02 |

<Info>
More job types are being added. Check the [job types overview](/job-types/overview) for the latest availability and pricing.
</Info>

If your balance is too low, the API returns:

```json
{
  "error": {
    "code": "INSUFFICIENT_CREDITS",
    "message": "Not enough credits for this job"
  }
}
```

HTTP status: `402`

## Usage tracking

The API provides usage data per organization:

```bash
curl "https://api.rendobar.com/billing/usage?start=2026-02-01&end=2026-02-09" \
  -H "Authorization: Bearer rb_live_YOUR_KEY"
```

Usage is tracked daily by job type, including job count, amount charged, and compute seconds.
