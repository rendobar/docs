The `raw.ffmpeg` job type lets you execute arbitrary FFmpeg commands through the Rendobar API. This gives you full FFmpeg power while Rendobar handles file transfer, execution isolation, timeout enforcement, and output upload.

## How it works

<Steps>

<Step title="Validate">
  Your command is validated against 8 security layers including a flag whitelist, format blocklist, and filter blocklist.
</Step>

<Step title="Download">
  Input URLs in your command are downloaded to a sandboxed execution environment.
</Step>

<Step title="Substitute">
  URLs are replaced with local file paths before execution.
</Step>

<Step title="Execute">
  The FFmpeg command runs inside an isolated container with network access disabled.
</Step>

<Step title="Upload">
  The output is uploaded to R2 storage.
</Step>

<Step title="Return">
  A presigned download URL is available via `GET /jobs/:id`.
</Step>

</Steps>

## Submitting a raw FFmpeg job

Submit via `POST /jobs` with a real FFmpeg command. Put input URLs directly in `-i` flags:

```bash
curl -X POST https://api.rendobar.com/jobs \
  -H "Authorization: Bearer rb_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "raw.ffmpeg",
    "inputs": {},
    "params": {
      "command": "ffmpeg -i https://example.com/video.mp4 -i https://example.com/audio.mp3 -c:v libx264 -c:a aac -map 0:v -map 1:a output.mp4"
    }
  }'
```

### Request fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Must be `"raw.ffmpeg"` |
| `inputs` | Record\<string, string\> | Yes | Additional named inputs (use `{}` when URLs are in the command) |
| `params.command` | string | Yes | Real FFmpeg command with input URLs in `-i` positions |
| `params.outputFormat` | string | No | Override output format (default: inferred from trailing filename, or `"mp4"`) |
| `params.timeout` | number | No | Max execution time in seconds (1-900, default 120) |

### Output format

The output format is inferred from the trailing filename in your command (e.g., `output.mp4` â†’ MP4, `output.webm` â†’ WebM). You can override it with `params.outputFormat`.

## Validation endpoint

You can validate a command without executing it (free, no auth required):

```bash
curl -X POST https://api.rendobar.com/ffmpeg/validate \
  -H "Content-Type: application/json" \
  -d '{ "command": "ffmpeg -i video.mp4 -vf scale=1280:720 output.mp4" }'
```

Returns `{ data: { valid: true, args: [...], inferredOutputFormat: "mp4" } }` or `{ data: { valid: false, error: "..." } }`.

## Allowed flags

Only whitelisted FFmpeg flags are permitted (~120 flags). Common ones include:

| Flag | Purpose |
|------|---------|
| `-i` | Input file |
| `-f` | Force format |
| `-c:v`, `-c:a` | Video/audio codec (stream specifiers like `-c:v:0` supported) |
| `-vf` | Video filter |
| `-af` | Audio filter |
| `-filter_complex` | Complex filter graph |
| `-map` | Stream mapping |
| `-ss` | Seek position |
| `-t` | Duration |
| `-r` | Frame rate |
| `-s` | Resolution |
| `-b:v`, `-b:a` | Video/audio bitrate |
| `-crf` | Constant rate factor |
| `-preset` | Encoding preset |
| `-an`, `-vn` | Disable audio/video |
| `-y` | Overwrite output |
| `-movflags` | MOV/MP4 flags |
| `-strict` | Strict standards compliance |

Any unrecognized flag causes the job to be rejected with a validation error.

## Security model

Rendobar applies 8 security layers to raw FFmpeg jobs:

1. **Clean environment.** All inherited env vars are stripped. Only `PATH`, `HOME`, `TMPDIR` remain.
2. **Flag whitelist.** ~120 known-safe flags permitted, including stream specifiers.
3. **Format blocklist.** Dangerous `-f` values blocked: `lavfi`, `concat`, `hls`, `tee`, `segment`, `dash`, `null`.
4. **Filter function blocklist.** File-reading filters blocked: `movie`, `amovie`, `sendcmd`, `zmq`, `ladspa`, `frei0r`.
5. **Filter parameter blocklist.** `textfile=`, `filename=` in subtitles, absolute paths in subtitle filters.
6. **Pattern blocklist + URL extraction.** Protocols, path traversal (`../`), `/proc/`, `/sys/`, `subfile:` blocked.
7. **Per-input protocol restriction.** `-protocol_whitelist file` injected before each `-i` server-side.
8. **Container isolation.** Trigger.dev container sandboxing with restricted stdio.

## Cost

Raw FFmpeg jobs are priced based on compute time. Compute time is measured from when FFmpeg starts to when it finishes, not including file transfer.

## Timeout limits

| Plan | Max timeout |
|------|-------------|
| Free | 5 minutes |
| Pro | 15 minutes |

If your FFmpeg command exceeds the timeout, the job fails with a `PROVIDER_TIMEOUT` error and no credits are deducted.

## Examples

All examples use `POST /jobs` with `type: "raw.ffmpeg"`.

### Scale a video to 720p

```json
{
  "type": "raw.ffmpeg",
  "inputs": {},
  "params": {
    "command": "ffmpeg -i https://example.com/video.mp4 -vf scale=1280:720 -c:v libx264 -preset fast -crf 23 output.mp4",
}
}
```

### Extract audio from video

```json
{
  "type": "raw.ffmpeg",
  "inputs": {},
  "params": {
    "command": "ffmpeg -i https://example.com/video.mp4 -vn -c:a aac -b:a 128k output.m4a",
}
}
```

### Trim a video (30 seconds starting at 1 minute)

```json
{
  "type": "raw.ffmpeg",
  "inputs": {},
  "params": {
    "command": "ffmpeg -i https://example.com/video.mp4 -ss 00:01:00 -t 00:00:30 -c:v libx264 -c:a aac output.mp4",
}
}
```

### Merge video and audio tracks

```json
{
  "type": "raw.ffmpeg",
  "inputs": {},
  "params": {
    "command": "ffmpeg -i https://example.com/video.mp4 -i https://example.com/narration.mp3 -c:v libx264 -c:a aac -map 0:v -map 1:a output.mp4",
}
}
```

## Error handling

If FFmpeg exits with a non-zero status code, the job fails with a `PROVIDER_ERROR`:

```json
{
  "error": {
    "code": "PROVIDER_ERROR",
    "message": "FFmpeg process exited with code 1"
  }
}
```

If the command contains disallowed flags, it is rejected before dispatch:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Flag '-protocol_whitelist' is not allowed in raw FFmpeg commands"
  }
}
```
