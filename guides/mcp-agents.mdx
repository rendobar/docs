Connect any AI agent to `api.rendobar.com/mcp` and it can submit jobs, poll status, upload files, and check the account balance via five typed MCP tools — no HTTP code, no SDK.

This page is the task-focused guide: how to connect, how a full workflow runs. For the protocol reference, tool schemas, and supported agents list, see the [MCP overview](/mcp/overview) and [tools reference](/mcp/tools-reference).

## Connect Claude Desktop or Claude Code

Add Rendobar to your MCP config:

```json
{
  "mcpServers": {
    "rendobar": {
      "url": "https://api.rendobar.com/mcp",
      "headers": {
        "Authorization": "Bearer rb_live_YOUR_KEY"
      }
    }
  }
}
```

Any MCP-compatible agent can connect the same way over the Streamable HTTP transport.

## Example: end-to-end video workflow

<Steps>

<Step title="Check balance">
  Call `get_account` to verify there are enough credits before running an expensive job.
</Step>

<Step title="Upload (if needed)">
  Call `upload_media` with the local filename to get a temporary upload endpoint, then POST the file there. The response includes a `downloadUrl` to pass into `submit_job`.
</Step>

<Step title="Submit the job">
  Call `submit_job` with `type: "raw.ffmpeg"`, the inputs map, and the FFmpeg command.
</Step>

<Step title="Poll for completion">
  Call `get_job` with the returned `jobId` until `status: "complete"`. The response then contains the signed `outputUrl`.
</Step>

<Step title="Report the result">
  Hand the output URL and metadata back to the user.
</Step>

</Steps>

## Authentication

Pass your API key in the `Authorization` header — the same key you use for the REST API works for MCP. MCP clients that speak OAuth 2.1 natively can use that path instead. See [Authentication](/authentication) for key management and the OAuth flow.
