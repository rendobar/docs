Point any MCP-compatible agent at `https://api.rendobar.com/mcp`. It can submit jobs, poll, upload files, and check the balance through five typed tools — no HTTP code, no SDK.

For the protocol reference and tool schemas: [MCP overview](/mcp/overview), [tools reference](/mcp/tools-reference).

## Connect Claude Desktop or Claude Code

```json
{
  "mcpServers": {
    "rendobar": {
      "url": "https://api.rendobar.com/mcp",
      "headers": {
        "Authorization": "Bearer rb_live_YOUR_KEY"
      }
    }
  }
}
```

Any MCP-compatible agent connects the same way over the Streamable HTTP transport.

## End-to-end workflow

The agent runs five tool calls to upload a file, render it, and report back:

1. **`get_account`** — confirm credit balance covers the estimated cost
2. **`upload_media`** — get a signed upload endpoint for the local file, then `POST` it there. The response includes a `downloadUrl` for the next step
3. **`submit_job`** — `type: "raw.ffmpeg"`, `inputs` map, `params.command`
4. **`get_job`** — poll until `status: "complete"`. The response has `outputUrl`
5. Hand the result back to the user

## Authentication

API key in the `Authorization` header. Same keys you use for REST. MCP clients with native OAuth 2.1 can run a PKCE flow against Rendobar's authorization server instead — see [Authentication](/authentication).
