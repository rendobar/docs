Credits are the universal billing unit for all Rendobar jobs. Every operation costs credits based on the job type and input/output duration.

## Credit lifecycle

<Steps>

<Step title="Pre-check">
  When you submit a job, Rendobar checks that your balance covers the estimated cost. No credits are deducted at this stage. If your balance is too low, the API returns `402 INSUFFICIENT_CREDITS`.
</Step>

<Step title="Execution">
  The job runs. Your balance is not touched during processing.
</Step>

<Step title="Debit">
  On successful completion, the actual cost is calculated (based on real output duration or compute time) and atomically deducted from your balance.
</Step>

<Step title="Refund">
  If a job fails or is cancelled, no credits are charged. Failed jobs never cost anything.
</Step>

</Steps>

## Cost models

Different job types use different cost models:

| Cost Model | How it works | Example job types |
|-----------|-------------|-------------------|
| Per minute (output) | Credits scale with output video duration | watermark.apply, transcode, caption.burn |
| Per minute (input) | Credits scale with input video duration | caption.extract, compliance.gdpr.redact |
| Flat rate | Fixed credit cost regardless of duration | extract.metadata, qr.generate, compliance.moderate (image) |
| Per compute time | Credits based on actual processing time | raw.ffmpeg |

## Checking your balance

```bash
curl https://api.rendobar.com/billing/balance \
  -H "Authorization: Bearer rb_live_YOUR_KEY"
```

```json
{
  "data": {
    "balance": 5000000000,
    "rollover": 0
  }
}
```

<Info>
The `balance` field is in nanodollars (1,000,000,000 = $1.00). The dashboard displays this as a formatted dollar amount.
</Info>
